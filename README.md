# AI Ticket Classifier — n8n workflow (тестовое задание)

<img width="1440" height="764" alt="Workflow" src="https://github.com/user-attachments/assets/e801da70-8a47-4080-97e5-4f450e32750a" />
Telegram-бот, который принимает обращение пользователя, отправляет его в LLM (GigaChat) и сам определяет: категорию, приоритет, кто должен этим заниматься, и краткую суть проблемы. Дальше сам рассылает тикет нужному специалисту в отдельный чат.

## Как это работает

1. Пользователь пишет боту.
2. Обращение уходит в GigaChat с function calling — модель возвращает ответ строго по заданной схеме (category, priority, summary, assignee, confidence).
3. Если ответ неполный — уходит алерт в DevOps.
4. Если модель сама не уверена в классификации (confidence < 0.7) — бот не шлёт тикет, а переспрашивает пользователя.
5. Если всё ок — тикет летит нужному специалисту: Support, Sales, DevOps или Account Manager.

## Узлы workflow

| Узел | Что делает |
|---|---|
| **Webhook** | Принимает сообщение из Telegram |
| **Set** | Хранит токены и ID чатов специалистов |
| **New users control** | Проверяет, это `/start` или обычное сообщение |
| **Send greetings** | Приветствие для нового пользователя |
| **Generate uuid** | Генерирует ID запроса (нужен для авторизации в GigaChat) |
| **Set token** | Получает OAuth-токен GigaChat |
| **Get a prompt** | Забирает system-промпт из Google Docs |
| **Create request** | Отправляет обращение в GigaChat, получает классификацию |
| **Response control** | Проверяет, что модель вернула все нужные поля |
| **Confidence control** | Проверяет уверенность модели — если низкая, просит пользователя уточнить |
| **Switch** | Определяет, кому маршрутизировать тикет, по полю assignee |
| **Send support / sales / devops / account manager ticket** | Отправляет тикет нужному специалисту в Telegram |
| **Send alert about token / prompt / request** | Если что-то из этого сломалось — пишет в чат DevOps, что именно упало |
| **Bad response** | Если ответ модели неполный — сообщает в DevOps |
| **Register webhook** | Привязывает бота к вебхуку n8n (разовая настройка) |

## Промпт в Google Docs

System-промпт вынесен в отдельный Google Docs. Правки формулировок и инструкций для модели подхватываются на следующем же сообщении, без захода в n8n.

## Function calling (classify_ticket)

Классификация реализована через function calling: модели передаётся схема функции с перечислимыми (enum) значениями для category, priority и assignee. Модель физически не может вернуть значение, которого нет в списке. В описание функции добавлено 5 примеров с готовыми ответами — они задают модели границы: где приоритет высокий, а где нет, кому уходит какая проблема.

## Почему GigaChat

GigaChat взят для тестового задания из практических соображений — у него 1 000 000 бесплатных токенов, этого с запасом хватает на прототип и проверку. Для продакшена разумнее взять модель с прозрачными бенчмарками (OpenAI, Anthropic, DeepSeek и т.д.) — там понятно качество function calling и следования схеме, что для этой задачи критичнее, чем для генерации текста.

## Стек и инфраструктура

- **n8n** — поднят через `docker-compose`.
- **GigaChat** — LLM с function calling.
- **Telegram Bot API** — приём обращений и рассылка тикетов.
- **Google Docs API** — хранилище промпта.
- **nginx** — реверс-прокси перед n8n, терминирует внешние запросы и проксирует их на локальный порт n8n (`proxy_pass`). Поддержаны два варианта TLS:
  - **Let's Encrypt** — сертификат выпускается и обновляется через certbot, TLS терминируется на самом nginx;
  - **Cloudflare** — TLS терминируется на стороне Cloudflare (режим Full/Strict до источника), на сервере nginx слушает локально по HTTP и проксирует на n8n; сертификат на origin-сервере при этом либо self-signed, либо Cloudflare Origin Certificate.

  Оба варианта закрывают вебхук HTTPS-эндпоинтом, который требует Telegram; выбор не влияет на логику самого workflow.

## Запуск

1. Поднять n8n через `docker-compose`.
2. Настроить nginx + домен (Let's Encrypt или Cloudflare — на выбор).
3. Импортировать `AI_bot_cinema.json` в n8n.
4. В узле **Set** заполнить: токен бота, ключ GigaChat, ID чатов Support/Sales/DevOps/Account Manager.
5. В узле **Get a prompt** указать свой Google Docs и подключить Google OAuth2.
6. Активировать workflow, выполнить **Register webhook**.
7. Написать боту `/start`, потом — тестовое обращение.
